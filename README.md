# JUnit Guardian

**LLM-powered JUnit test scanner and generator for Java/Gradle projects.**  
Uses a local [Ollama](https://ollama.com) instance — no cloud API key needed.  
The Git pre-commit hook **warns** when coverage is low but never blocks your commit.

---

## How it works

```
git commit
    └── pre-commit hook fires
            └── JUnit Guardian scans src/main/java
                    ├── finds classes with no matching *Test.java
                    ├── calls Ollama (codellama / deepseek-coder / …)
                    │       └── generates complete JUnit 5 test classes
                    ├── writes generated files into src/test/java
                    └── prints coverage report  ⚠ warn if < 80% (commit proceeds)
```

## Quick start

### 1. Build the tool

```bash
cd junit-guardian
./gradlew shadowJar          # produces build/libs/junit-guardian-1.0.0.jar
```

### 2. Install into your Java project

```bash
bash scripts/install.sh /path/to/your-java-project
```

This will:
- Copy the pre-commit hook into `.git/hooks/`
- Write a `.guardian.env` config file
- Check that Ollama + model are available

### 3. Pull the Ollama model

```bash
ollama pull codellama        # default
# or
ollama pull deepseek-coder   # often better for Java
ollama pull qwen2.5-coder
```

### 4. Activate

```bash
source /path/to/your-project/.guardian.env
ollama serve &               # if not already running
```

### 5. Commit as normal

```bash
git add src/main/java/com/example/MyService.java
git commit -m "feat: add MyService"
# Guardian runs automatically, warns if tests are missing,
# and writes MyServiceTest.java into src/test/java.
```

## Environment variables

Set in `.guardian.env` (auto-generated by install.sh):

| Variable | Default | Description |
|----------|---------|-------------|
| `JUNIT_GUARDIAN_JAR` | *(auto-detected)* | Path to the fat JAR |
| `JUNIT_GUARDIAN_OLLAMA` | `http://localhost:11434` | Ollama base URL |
| `JUNIT_GUARDIAN_MODEL` | `codellama` | Model to use |
| `JUNIT_GUARDIAN_MODE` | `generate` | `scan` / `generate` / `report` |
| `JUNIT_GUARDIAN_MIN_COV` | `80` | Warning threshold (%) |

## CLI usage (standalone)

```bash
JAR=build/libs/junit-guardian-1.0.0.jar

# Scan only — print coverage report
java -jar $JAR --mode scan /path/to/project

# Generate missing tests
java -jar $JAR --mode generate /path/to/project

# Generate + write JSON report
java -jar $JAR --mode report /path/to/project

# Custom model / URL
java -jar $JAR \
  --llm-provider ollama \
  --ollama-url http://localhost:11434 \
  --model deepseek-coder \
  --min-coverage 90 \
  /path/to/project

# Block commit if below threshold (CI mode)
java -jar $JAR --fail-on-missing --min-coverage 80 /path/to/project
```

Full help: `java -jar $JAR --help`

---

## Modes explained

| Mode | What it does |
|------|-------------|
| `scan` | Prints a coverage report. No files written. |
| `generate` | Writes `*Test.java` files for every uncovered class. |
| `report` | `generate` + writes `junit-guardian-report.json`. |

## Generated test quality

The LLM prompt instructs the model to:

- Cover every **public and package-private method**
- Write **happy path**, **edge case**, and **exception** tests
- Use **Mockito** to mock collaborators
- Use **AssertJ** fluent assertions
- Follow the **Arrange / Act / Assert** pattern
- Target **≥80% branch coverage**

Add these dependencies to your project's `build.gradle` to compile the generated tests:

```groovy
testImplementation 'org.junit.jupiter:junit-jupiter:5.10.2'
testImplementation 'org.mockito:mockito-junit-jupiter:5.11.0'
testImplementation 'org.assertj:assertj-core:3.25.3'
testRuntimeOnly    'org.junit.platform:junit-platform-launcher'
```

## Recommended Ollama models for Java

| Model | Pull command | Notes |
|-------|-------------|-------|
| `deepseek-coder` | `ollama pull deepseek-coder` | Best Java quality |
| `qwen2.5-coder` | `ollama pull qwen2.5-coder` | Fast + accurate |
| `codellama` | `ollama pull codellama` | Default, widely available |
| `mistral` | `ollama pull mistral` | Good general purpose |

Change the model in `.guardian.env`:
```bash
export JUNIT_GUARDIAN_MODEL="deepseek-coder"
```

## Project structure

```
junit-guardian/
├── build.gradle                  ← Gradle build (fat JAR via Shadow plugin)
├── settings.gradle
├── scripts/
│   ├── pre-commit                ← Git hook (copy to .git/hooks/)
│   └── install.sh                ← One-command installer
└── src/main/java/com/junitguardian/
    ├── Main.java                 ← CLI entry point (picocli)
    ├── Guardian.java             ← Orchestrator
    ├── GuardianConfig.java       ← Config value object
    ├── ConsoleLogger.java        ← Coloured terminal output
    ├── llm/
    │   ├── LlmClient.java        ← Interface
    │   ├── LlmClientFactory.java
    │   ├── OllamaLlmClient.java  ← Ollama implementation
    │   ├── AnthropicLlmClient.java
    │   ├── OpenAiLlmClient.java
    │   └── PromptBuilder.java    ← System + user prompts
    ├── model/
    │   ├── SourceFile.java
    │   └── ScanReport.java
    └── scanner/
        ├── JavaSourceScanner.java      ← Walks src/main/java, parses AST
        └── TestCoverageAnalyzer.java   ← Matches sources → test files
```
